TGradientProgressBar control

About

This is the manual of TGradientProgressBar control, version 1.0, released on 23th July 2002. TGradientProgressBar is a shareware Delphi component written by Nikolaos Georgiou.

Description

TGradientProgressBar is a replacement for the standard Windows progress bar. It offers three different colorful appearances:

  1. Gradient fill. You select two colors for the progress bar and a rainbow-like fill is applied to the progress bar, shifting smoothly from one color to another.


  2. Changing color. Again, you select two colors. The progress bar’s color changes according to its position using an appropriate mix of the two colors.
  3. One color only. This is the standard progress bar. You can use this option if you don’t want anything fancy.

TGradientProgressBar descends directly from TGraphicControl thus it uses less resources than the standard Windows progress bar which is a windowed control.

 

Properties

property BackColor: TColor;

BackColor is used to fill the interior of the control.

property BorderColor: TColor;

BorderColor is used to paint the border of the control when BorderStyle is set to bsFlat.

property BorderStyle: TGPBBorderStyle;

type TGPBBorderStyle = (bsNone, bsRaised, bsLowered, bsFlat, bsSpace);

BorderStyle is used to select the appearance of the border. The following values are used:

  • bsNone. No border will be painted and the progress bar will occupy the entire client area of the control.
  • bsRaised. A raised border will be painted.
  • bsLowered. A lowered border will be painted.
  • bsFlat. A flat border will be painted, using the color specified by BorderColor.
  • bsSpace. No border will be painted but the progress bar will be limited by BorderWidth.

property BorderWidth: Integer;

BorderWidth specified the amount of empty space that will seperate the progress bar from its border.

property ColorOption: TGPBColorOption;

TGPBColorOption = (coGradient, coChangingColor, coOneColor);

ColorOption specifies the appearance of the control. The following illustration shows how the TGradientProgressBar control appears in each possible value of ColorOption.

property Direction: TGPBDirection;

TGPBDirection = (diHorizontal, diVertical);

Direction is a property that controls the appearance of TGradientProgressBar. It has two possible values:

  • diHorizontal. Progress advances from left to right.
  • diVertical. Progress advances from bottom to top.

property Min: Integer;

The minimum value of the control’s position. It must be less than Max or an exception will be raised.

property MinColor: TColor;

The color used to indicate the minimum value. It is used in conjunction with MaxColor to make all possible colors when ColorOption is either coGradient or coChangingColor.

property Max: Integer;

The maximum value of the control’s position. It must be greater than Min or an exception will be raised.

property MaxColor: TColor;

The color used to indicate the maximum value. See MinColor.

property Position: Integer;

The progress’ current position. It must be greater or equal than Min and less or equal than Max. If these bounds are violated, an exception will be raised.

property Step: Integer;

The amount that will be added to Position by a call to StepIt. If you enter a negative value, StepIt will decrease Position.

 

Methods

procedure StepIt;

Increments Position by Step (or decreases it if Step is negative). If this variation causes Position to break the limits of Min or Max an exception will not be raised. Instead, the value will be adjusted to fit in the bounds in a circular manner.
